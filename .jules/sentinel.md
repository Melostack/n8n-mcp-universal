## 2026-04-12 - [Remove Sensitive Data Logging in HTTP Server]\n**Vulnerability:** The HTTP server endpoints logged raw request bodies (including JSON-RPC parameters) and entire request headers to the standard logger, which could expose sensitive user data, API keys, and Bearer tokens in logs.\n**Learning:** In MCP integrations, JSON-RPC 'params' might legitimately contain secrets that shouldn't persist in plaintext logs, and request headers definitely contain authorization tokens. We must explicitly scrub these payloads instead of simply dumping the request objects for debugging.\n**Prevention:** Only log safe request metadata (e.g., method, content length, 'hasBody', URL) rather than full object serialization for potentially untrusted or authenticated inputs.
