## 2024-05-24 - [SSRF Protection: Missing DNS Pinning]
**Vulnerability:** The `validateWebhookUrl` method in `SSRFProtection` performed DNS resolution to check for internal IP addresses, but returned only a boolean. Axios would then perform its own DNS resolution. This created a Time-of-Check to Time-of-Use (TOCTOU) vulnerability where an attacker could exploit DNS rebinding (changing the DNS record to point to an internal IP after the check but before the fetch).
**Learning:** Checking an IP against a blocklist requires that the exact resolved IP is used for the subsequent request. Merely validating the hostname and letting the HTTP client resolve it again leaves the system vulnerable to DNS rebinding.
**Prevention:** The validation function must return the exact IP address resolved during the check, and the HTTP client (like Axios) must be configured to use this specific IP (via a custom `lookup` function or pinning mechanism) to prevent secondary, potentially malicious resolutions.
